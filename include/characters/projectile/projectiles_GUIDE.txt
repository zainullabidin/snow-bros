FILE 1: Projectile.h
----------------------------------------------------------------
OWNER        : Student B (Minahil)
DEPENDS ON   : Entity.h, SFML
NEEDED BY    : Snowball.h, Knife.h, ArtilleryRocket.h,
               CollisionDetector.h

WHAT IT IS:
Abstract base class for every projectile in the game.
Student B defines the interface. Student A implements
each specific projectile type.

PURE VIRTUAL METHODS:
    update(float dt)         — move projectile every frame
    draw(RenderWindow& w)    — draw projectile
    onHit()                  — what happens on collision
    getHitBox() : FloatRect  — collision rectangle

SHARED METHODS (Student B implements):
    isActive() : bool        — is projectile still in play?

PROTECTED MEMBERS:
    sf::Vector2f position
    sf::Vector2f velocity
    bool active  = true
    int  damage  = 1

WHAT IT DELIVERS AFTER IMPLEMENTATION:
A common interface so CollisionDetector can check
any projectile's hitbox the same way regardless
of whether it is a snowball, knife, or rocket.

SIMPLE EXPLANATION:
Projectile.h is the template that says every flying
object in the game must know how to move, draw
itself, and react when it hits something.

RULES FOR STUDENT B:
- Keep this abstract. No specific projectile logic here.
- onHit() must be pure virtual so each type handles
  its own collision response differently.

FILE 2: Snowball.h
----------------------------------------------------------------
OWNER        : Student A
DEPENDS ON   : Projectile.h, SFML
NEEDED BY    : SnowballLauncher.h, CollisionDetector.h,
               LevelManager.h

WHAT IT IS:
The most important and most complex class in the
entire project. The core mechanic of Snow Bros lives here.
Get this right and the game feels good.
Get it wrong and nothing else matters.

THREE STATES (this is the heart of the mechanic):

    STATE 1 — FLYING:
        Just thrown by player.
        Travels horizontally in one direction.
        Wraps around screen edges (exits right, enters left).
        On contact with enemy: increments enemy's snowCover.
        If snowCover reaches max the enemy becomes encased.
        Partially hit enemy is slowed but still dangerous.

    STATE 2 — ENCASED (rollable):
        Enemy is fully covered in snow.
        Snowball sits on the enemy waiting to be kicked.
        Player walks into it to start rolling.
        Affected by gravity — falls off platforms.
        Grows in size with each enemy it kills (grow()).
        Each kill fires CHAIN_KILL event via EventBus.

    STATE 3 — SHAKE FREE:
        If left alone a shake timer counts down.
        When timer hits zero enemy breaks free.
        Snowball disappears, enemy resumes normal behavior.

STATE DIAGRAM:
    FLYING ──(hits enemy, snowCover max)──► ENCASED
                                                │
                    ┌───────────────────────────┤
                    │                           │
                    ▼                           ▼
              (player kicks)            (timer expires)
                    │                           │
                    ▼                           ▼
                ROLLING                   ENEMY BREAKS FREE
                    │
              (hits enemies)
                    │
                    ▼
              CHAIN REACTION

ADDITIONAL MEMBERS:
    int   snowLevel    — current snow state (0=flying,1=partial,2=full)
    bool  rolling      — is snowball currently rolling?
    float rollSize     — visual size multiplier (grows with chain)
    float shakeTimer   — countdown until enemy breaks free
    int   chainCount   — how many enemies hit while rolling

KEY METHODS:
    update(dt)         — manages state transitions every frame
    draw(window)       — draws snowball at current state and size
    onHit()            — handles hitting an enemy
    startRolling()     — transitions from ENCASED to ROLLING
    grow()             — increases size after each chain kill
    isRolling() : bool
    getChainCount() : int

SCREEN WRAPPING:
    if position.x > SCREEN_WIDTH  → position.x = 0
    if position.x < 0             → position.x = SCREEN_WIDTH
    Only applies in FLYING state. Rolling snowballs do not wrap.

WHAT IT DELIVERS AFTER IMPLEMENTATION:
The complete Snow Bros core mechanic. Player can throw,
encase enemies, kick encased enemies, create chain
reactions, and enemies can shake free if not collected.

SIMPLE EXPLANATION:
Snowball starts as a weapon, becomes a trap, then
becomes a rolling wrecking ball that can take out
multiple enemies at once for bonus points.

RULES:
- Plan the state machine on paper before coding.
  This is the most complex state logic in the project.
- shakeTimer should start at around 3 seconds.
  This gives the player time to kick without being
  too forgiving.
- rollSize starts at 1.0 and grows by 0.2 per kill.
  Cap it at 3.0 so it does not become absurd.
- Always fire CHAIN_KILL event for every enemy killed
  while rolling. ScoreSystem applies chain bonus.

FILE 3: Knife.h
----------------------------------------------------------------
OWNER        : Student A
DEPENDS ON   : Projectile.h, SFML
NEEDED BY    : Tornado.h, CollisionDetector.h

WHAT IT IS:
A straight-line projectile thrown by Tornado at the
player's last known position. Simple and dangerous.

ADDITIONAL MEMBERS:
    sf::Vector2f direction   — normalized direction vector
                               calculated once at spawn time

KEY BEHAVIOR:
    Direction is calculated at the moment of creation:
        direction = normalize(playerPosition - tornadoPosition)
    
    Every frame:
        position += direction * speed * dt
    
    Deactivates when:
        - hits the player (fires PLAYER_HIT event)
        - reaches screen edge

WHAT IT DELIVERS AFTER IMPLEMENTATION:
Tornado can now attack the player from a distance.
Player must dodge by moving after the knife is thrown.

SIMPLE EXPLANATION:
Knife is like an arrow. Direction is locked at the
moment of firing. It does not follow the player.
Move after it is thrown and you are safe.

RULES:
- Direction is FIXED at creation time. Never update it.
- Always normalize the direction vector or speed
  will vary based on distance from player.
- On hitting player fire PLAYER_HIT via EventBus.
  Never remove a life directly — that is Player's job.

FILE 4: ArtilleryRocket.h
----------------------------------------------------------------
OWNER        : Student A
DEPENDS ON   : Projectile.h, SFML
NEEDED BY    : Gamakichi.h, CollisionDetector.h

WHAT IT IS:
Rockets fired by Gamakichi in multiple directions.
The most visually dramatic projectile. Explodes on impact
creating a blast zone that damages the player.

ADDITIONAL MEMBERS:
    bool  exploding      — is rocket currently exploding?
    float blastRadius    — size of explosion area
    float explosionTimer — how long explosion lasts

TWO-STAGE BEHAVIOR:

    STAGE 1 — FLYING:
        Moves in launch direction at high speed.
        Checks for collision with platforms and floor.
        On contact → switches to EXPLODING state.

    STAGE 2 — EXPLODING:
        Stops moving.
        Draws expanding explosion circle.
        Checks if player is inside blast radius.
        If player is inside → fires PLAYER_HIT event.
        After explosionTimer expires → deactivates.

KEY METHODS:
    update(dt)       — manages both stages
    draw(window)     — draws rocket or explosion
    onHit()          — triggers explosion
    explode()        — private, starts explosion sequence
    isExploding() : bool

WHAT IT DELIVERS AFTER IMPLEMENTATION:
Gamakichi can fire multiple rockets that explode on
landing creating dangerous zones the player must avoid.
Combined with multiple phases this makes the final
boss fight genuinely challenging.

SIMPLE EXPLANATION:
ArtilleryRocket is like a grenade that flies through
the air and explodes when it lands. The explosion
itself is dangerous — not just the direct hit.

RULES:
- Explosion must last long enough to be visible
  and dangerous. Suggest 0.5 to 0.8 seconds.
- blastRadius should be proportional to the game's
  scale. Suggest 80 to 100 pixels radius.
- Multiple rockets can be active simultaneously.
  Gamakichi fires them in bursts.
